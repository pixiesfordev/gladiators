Grid Framework  3.0.0
Easy grid setup and scripting for Unity 3D
Upgrading from 2.x to 3.x

Grid Framework version 3.0 is a major release and as such not backwards compatible with the earlier version 2 releases. The biggest change is the rewritten rendering system, which allows for different rendering backends, and the complete overhaul of the Playmaker support.

Grid Famework has been broken up into one large core package and several smaller add-on packages. This more modular architecture means that instead of defining preprocessor symbols you can instead opt-in by installing a separate package.

Grids

The following are only relevant for people who have implemented their own grids.

  • Reworked the caching of member variables in grids: use MarkAsDirty instead of setting _isDirty manually, implement ComputeCachedMembers instead of UpdateCachedMembers.
  • Removed the Transform_ property, use Unity's own GameObject.transform instead.
  • Removed the various events when a grid property is changed.
  • The CoordinateSystem enumeration type no longer exist on the various grid classes. See the next point.
  • The extension methods NearestVertex, NearestFace and NearestCell have been replaced with Nearest___Vertex, Nearest___Face and Nearest___Cell spectively, where the infix ___ stands for the coordinate system.

Base grids

The abstract parent class of all grids, Grid, has been renamed to BaseGrid. Unity has its own Grid class, and while both can co-exist due to being in different namespaces, the editor would print an annoying warning every time.

The rendering system

Grid rendering at runtime has been completely redone: renderers work as before, but now there are backends which actually display the result of the renderers. The grid camera backend corresponds to the old way of rendering grids.

Grid camera backend

The GridFramework.Rendering.GridCameraBackend component corresponds to the old GridCamera component and works the same.

Vectrosity support

Support for Vectrosity is now implemented as one of the rendering backends. It works the same as before, but the new name of the component is GridFramework.Rendering.Vectrosity.VectrosityBackend.

Renderers

Renderers use fewer state variables and prefer returning concrete values instead.

  • GridRenderer.CountLines needs to return a tuple of line counts instead of mutating an instance variable.
  • GridRenderer.ComputeLines takes an array of preallocated points as argument. Implementations are expected to mutate those points instead of mutating the points of an instance variable.
  • GridRenderer.ComputeLines returns an array of arrays of line segments instead of an array of arrays of arrays of Vector3

Playmaker actions

Playmaker actions are a separate package. All actions have been completely overhauled to be simpler to use.

All the separate getter and setter actions have been united into fewer and larger getter and setter actions. For example, if you wanted to set the From and To of a Parallelepiped renderer you had to use two separate actions. Now only one setter actions exists with two fields. If one of the fields is None then the value will not be changed.

Align panel

The align panel is a separate package.
